# Riepilogo Versione 0.0.3

**Datazione:** 15 Aprile 2026.
**Cambiamenti Principali (Audio Engine):**

- **Sostituzione Engine:** Installazione della suite nativa `kira` abbinata ai decodificatori interni (Symphonia) per gestire la riproduzione musicale slegata dal Web Audio API o HTML5 Audio tags del frontend, minimizzando l'impronta sulla CPU. *Patch post-rilascio:* Per azzerare i leak di RAM ed evitare che l'audio venga pre-caricato interamente in memoria, il buffer `StaticSoundData` è stato refactorizzato utilizzando nativamente formati streaming in `audio.rs`.
- **Creazione Bus Virtuali:** Impostazione architetturale del modulo `audio.rs` per ospitare il Channel della "Musica", incapsulando dinamicamente la risorsa.
- **Auto-Ducking Implementato:** Scritto il comando Rust che abbassa o alza fluidamente (-15dB/0dB con sub-millisecond precision tweening) la base in backgorund reagendo allo switch o pressione del microfono live. Comando bindato via IPC (`toggle_mic`).

**Stato di Completamento Architetturale:**
Sia il DB (Fase 2) che il motore Audio (Fase 3) sono pronti a ricevere chiamate dal frontend.
